| Composant | Statut | Détails |
|-----------|--------|---------|

## 2. Scénario de Test End-to-End

### Phase 0 — Prérequis & Installation

```bash
# 1. Cloner le repo
git clone https://github.com/tebeerr/Automated-Web-Penetration-Testing-Platform.git
cd Automated-Web-Penetration-Testing-Platform

# 2. Copier la config
cp .env.example .env
# Éditer .env : mettre JWT_SECRET à une vraie valeur aléatoire

# 3. Backend
cd apps/backend
python -m venv .venv
source .venv/bin/activate
pip install -e .[dev]
cp ../../.env .
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# 4. Frontend (nouveau terminal)
cd apps/frontend
npm install
npm run dev
```

**Vérification rapide :**
- `http://localhost:8000/docs` → Swagger UI doit s'afficher
- `http://localhost:8000/api/health` → `{"status":"ok","version":"..."}`
- `http://localhost:5173` → Dashboard React doit s'afficher

### Phase 2 — Test de la Protection SSRF

**Test 2.1 : URL valide**
```bash
curl -X POST http://localhost:8000/api/scans/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"target_url":"https://example.com","scan_profile":"full"}'
```
→ Attendu : `202 Accepted` avec un `ScanResponse`

**Test 2.2 : URL localhost (doit être bloquée)**
```bash
curl -X POST http://localhost:8000/api/scans/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"target_url":"http://127.0.0.1","scan_profile":"full"}'
```
→ Attendu : `400` ou `422` — SSRF bloqué

**Test 2.3 : URL IP privée (doit être bloquée)**
```bash
curl -X POST http://localhost:8000/api/scans/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"target_url":"http://192.168.1.1","scan_profile":"full"}'
```
→ Attendu : Rejeté par le SSRF guard

**Test 2.4 : Schéma non-HTTP**
```bash
curl -X POST http://localhost:8000/api/scans/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"target_url":"ftp://example.com","scan_profile":"full"}'
```
→ Attendu : Rejeté (seuls http/https acceptés)

**Test 2.5 : Cloud metadata (169.254.169.254)**
```bash
curl -X POST http://localhost:8000/api/scans/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"target_url":"http://169.254.169.254/latest/meta-data/","scan_profile":"full"}'
```
→ Attendu : Bloqué explicitement

- `SQLiProbe` → A03 (injection SQL réfléchie)
- `XSSProbe` → A03 (XSS réfléchi)

## 3. Base de Données — phpMyAdmin, oui ou non ?

### Réponse courte : **phpMyAdmin est inutile pour ce projet.**

### Pourquoi ?

phpMyAdmin est une interface web pour **MySQL/MariaDB uniquement**. Sentinel utilise **SQLite par défaut** et est conçu pour basculer vers **PostgreSQL** en production. À aucun moment MySQL n'entre dans l'architecture.

| Critère | phpMyAdmin | Sentinel |
|---------|-----------|----------|
| Base supportée | MySQL / MariaDB uniquement | SQLite (dev) / PostgreSQL (prod) |
| Compatible ? | ❌ Non | — |
| Utilité ? | ❌ Aucune | — |

### Ce que tu peux utiliser à la place

**En développement (SQLite) :**
- **DB Browser for SQLite** (GUI desktop) — `sudo apt install sqlitebrowser`
- **SQLite CLI** — `sqlite3 sentinel.db` puis `.tables`, `.schema`, `SELECT * FROM scans;`
- **VS Code extension** : "SQLite Viewer" ou "SQLite3 Editor"

**En production (PostgreSQL) :**
- **pgAdmin 4** — l'équivalent de phpMyAdmin pour PostgreSQL, interface web riche
- **DBeaver** — client universel (supporte SQLite ET PostgreSQL)
- **Adminer** — alternative légère web, supporte PostgreSQL

## 4. Comment Intégrer PostgreSQL (Migration SQLite → Postgres)

L'architecture est déjà prête, c'est un changement d'une seule variable.

### Étape 1 — Installer PostgreSQL
```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo -u postgres createuser sentinel_user -P    # définir un mot de passe
sudo -u postgres createdb sentinel_db -O sentinel_user
```

### Étape 2 — Installer le driver Python
```bash
pip install asyncpg
```

### Étape 3 — Modifier `.env`
```env
# Avant (SQLite)
DATABASE_URL=sqlite+aiosqlite:///./sentinel.db

# Après (PostgreSQL)
DATABASE_URL=postgresql+asyncpg://sentinel_user:MOT_DE_PASSE@localhost:5432/sentinel_db
```

### Étape 4 — Redémarrer le backend
```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```
→ SQLAlchemy va créer les tables automatiquement dans PostgreSQL (les modèles utilisent des types portables).

### Étape 5 — (Optionnel) Installer pgAdmin pour visualiser
```bash
# Via Docker
docker run -p 5050:80 \
  -e PGADMIN_DEFAULT_EMAIL=admin@sentinel.local \
  -e PGADMIN_DEFAULT_PASSWORD=admin \
  dpage/pgadmin4
```
→ Ouvrir `http://localhost:5050`, connecter au serveur PostgreSQL local.
